# <a name="azure-white-papers-about-mainframe-topics"></a>White papers do Azure sobre tópicos de mainframe

Obtenha recursos sobre a migração de mainframe, nova hospedagem e movimentação de sistemas com base em IBM Z para o Microsoft Azure.

### <a name="demystifying-mainframe-to-azure-migrationhttpsazuremicrosoftcomresourcesdemystifying-mainframe-to-azure-migration"></a>[Desmistificando mainframe para a migração do Azure](https://azure.microsoft.com/resources/demystifying-mainframe-to-azure-migration/)

O Azure fornece uma alternativa de mainframe capaz de oferecer recursos e funcionalidade equivalente. Neste documento, por Larry Mead da equipe do AzureCAT, aborda os principais componentes de mainframe IBM z/OS e os equivalentes do Azure. Ele também fornece um roteiro para iniciar a conversa com tomadores de decisões IT que assinaram filosofias de mainframe desatualizados.

### <a name="move-mainframe-compute-and-storage-to-azurehttpsazuremicrosoftcomresourcesmove-mainframe-compute-and-storage-to-azure"></a>[Mover o mainframe computação e armazenamento para o Azure](https://azure.microsoft.com/resources/move-mainframe-compute-and-storage-to-azure/)

Para executar cargas de trabalho de mainframe no Microsoft Azure, você precisa saber como os recursos do seu mainframe se comparam com o Azure. Com base em um z14 um mainframe IBM, este guia por Larry Mead do AzureCAT explora como obter resultados comparáveis no Azure. Os recursos de computação e armazenamento altamente escalonáveis no Azure fornecem benefícios imediatos para as organizações que executam cargas de trabalho de mainframe.

### <a name="microsoft-azure-government-cloud-for-mainframe-applicationshttpsazuremicrosoftcomresourcesmicrosoft-azure-government-cloud-for-mainframe-applications"></a>[Nuvem de governo do Microsoft Azure para aplicativos de mainframe](https://azure.microsoft.com/resources/microsoft-azure-government-cloud-for-mainframe-applications/)

Planejar uma migração de aplicativo é o momento ideal para adicionar valor e agilidade para até mesmo cargas de trabalho de mainframe bem estabelecidos. Neste guia rápido, Larry Mead do AzureCAT mostra como os agências governamentais dos Estados Unidos e seus parceiros podem usar Azure governamental para aplicativos de mainframe — e a migração não pode ser tão difícil quanto você acha. O Azure Government oferece as vantagens de um mainframe em um ambiente mais eficiente e ágil. Além disso, o Azure governamental acumulado uma autoridade provisória de funcionamento (P-ATO) para FedRAMP alto impacto.

### <a name="deploy-ibm-db2-purescale-on-azurehttpsazuremicrosoftcomresourcesdeploy-ibm-db2-purescale-on-azure"></a>[Implantar o IBM DB2 pureScale no Azure](https://azure.microsoft.com/resources/deploy-ibm-db2-purescale-on-azure/)

Saiba mais em nossa experiência com uma empresa que rehosted recentemente o seu ambiente do IBM DB2 no Azure. Este guia foi escrito por AzureCAT e DMJ membros da equipe que estavam lá: Larry Mead, Benjamin Guinebertière, Alessandro Vozza, and Jonathon Frost. Eles descrevem as etapas que levaram durante a migração. Suas descobertas foram revisadas por membros da equipe pureScale IBM Db2. Scripts de instalação, disponíveis no GitHub, baseiam-se na arquitetura do que a equipe usou para uma típica médias carga de trabalho OLTP.

### <a name="install-tmaxsoft-openframe-on-azurehttpsazuremicrosoftcomresourcesinstall-tmaxsoft-openframe-on-azure"></a>[Instalar o TmaxSoft OpenFrame no Azure](https://azure.microsoft.com/resources/install-tmaxsoft-openframe-on-azure/)

Modernize sua infraestrutura em escala de nuvem. TmaxSoft OpenFrame facilita a seus ativos de mainframe existentes de lift- and -shift-los para o Azure. Neste artigo, Read de Steve (AzureCAT) e Manoj Aerroju (TmaxSoft) explicam como configurar um ambiente OpenFrame adequado para desenvolvimento, demonstrações, testes e cargas de trabalho de produção.

### <a name="ibm-mainframe-to-microsoft-azure-reference-architecturehttpswwwastadiacomwhitepaperibm-mainframe-to-microsoft-azure"></a>[Mainframe IBM para arquitetura de referência do Microsoft Azure](https://www.astadia.com/whitepaper/ibm-mainframe-to-microsoft-azure)

Este white paper reflete que astadia 's mais de 25 anos de experiência de modernização de plataforma do mainframe. Eles explicam os benefícios e desafios dos esforços de modernização. Este guia fornece uma visão geral de mainframe IBM e uma arquitetura de referência de mainframe para o Azure da IBM. Ele também fornece uma visão a metodologia de sucesso Astadia.

### <a name="deploying-mainframe-applications-to-microsoft-azurehttpswwwmicrofocuscommediawhite-paperdeployingmainframeapplicationstomicrosoftazurewppdf"></a>[Implantando aplicativos de mainframe para o Microsoft Azure](https://www.microfocus.com/media/white-paper/deploying_mainframe_applications_to_microsoft_azure_wp.pdf)

Soluções do Micro Focus liberá-lo das restrições de mainframe proprietárias hardware e software. Neste guia, Micro Focus explica como implantar o COBOL e PL / eu aplicativos executados em mainframes IBM para a nuvem em vez disso.

### <a name="breathe-new-life-into-mainframeshttpswwwinfosyscommodernizationpagesbreathe-new-life-mainframesaspx"></a>[Dar vida nova aos mainframes](https://www.infosys.com/modernization/Pages/breathe-new-life-mainframes.aspx)

 Mainframes são cada vez mais desafiadores para empresas para manter. Este white paper Infosys e Microsoft destaca a estratégia vencedora para migração de mainframe com êxito. Casos de uso e exemplos ilustram as opções.
