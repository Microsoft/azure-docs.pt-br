# <a name="power-bi-app-storefront-details-tab"></a>Guia de detalhes da vitrine eletrônica de aplicativo do Power BI

Sobre o **nova oferta** página, use o **detalhes da vitrine eletrônica** guia para fornecer informações legais, vendas e marketing para seus clientes potenciais. Nessa guia você também pode configurar o gerenciamento de clientes potenciais que gera o Azure Marketplace. Essa forma longa é dividida em seis seções: **Detalhes da Oferta**, **Detalhes da Listagem**, **Artefatos de Marketing**, **Jurídico**, **Atendimento ao Cliente** e **Gerenciamento de Cliente Potencial**.  Um asterisco (*) no final de um rótulo de campo significa que o campo é obrigatório.


## <a name="offer-details-section"></a>Seção Detalhes da Oferta

No **detalhes da oferta** , digite as informações gerais sobre sua oferta do AppSource.

![Seção Detalhes da Oferta da guia Detalhes da Vitrine](./media/offer-details-section.png)

Para concluir os campos nesta seção, use a tabela a seguir:

|   Campo               |   DESCRIÇÃO                                                                           |
|-----------------------|-----------------------------------------------------------------------------------------|
| **Resumo da oferta**     | Uma breve finalidade do aplicativo, usando até 100 caracteres.                             |
| **Descrição da oferta** | Uma descrição do aplicativo, usando até 3.000 caracteres. Este campo dá suporte a formatação de HTML simples. |
|                       |                                                                                         |


## <a name="listing-details-section"></a>Seção Detalhes da Listagem

O **listando detalhes** seção reúne informações sobre o contexto do seu aplicativo: setores em que ele é normalmente usado, a categoria de aplicativo, produtos compatíveis e os termos de pesquisa associada.

![Seção Detalhes da Listagem da guia Detalhes da Vitrine](./media/listing-details-section.png)

Para concluir os campos nesta seção, use a tabela a seguir:
 
|   Campo                                  |   DESCRIÇÃO                                                        |
| --------------                           | ---------------------                                                |
| **Indústrias**                           | Selecione o setor de onde seu aplicativo se encaixa melhor. Se seu aplicativo se relaciona aos diversos setores, deixe esse campo em branco.      |
| **Categorias**                           | Selecione até três categorias que se relacionam ao seu aplicativo.     |
| **Link de ajuda para seu aplicativo**               | Forneça uma URL para uma página que oferece ajuda on-line para seu aplicativo.           |
| **Produtos com os quais seu aplicativo trabalha (máx. 3)** | Selecione o **New** sinal de adição para criar um campo de texto. No campo, insira o nome de um produto que seu aplicativo funciona com. Liste até três produtos.       |
| **Pesquisar palavras-chave (máx. de 3)**              | Insira até três palavras-chave que os usuários provavelmente usará para pesquisar para seu aplicativo no AppSource. Por exemplo, se o aplicativo é chamado de "Meu aplicativo enviando email", as palavras-chave podem ser **emails**, **mala direta**, e **aplicativo de email**. |
|  |  |


## <a name="marketing-artifacts-section"></a>Seção de artefatos de marketing

No **artefatos de Marketing** seção, upload de identidade visual e materiais para exibir no AppSource de marketing.  Esta seção é dividida em quatro subseções: **Logotipos**, **Vídeos**, **Documentos** e **Capturas de tela**. Logotipos e capturas de tela são os únicos necessários artefatos de marketing. Mas, para o melhor apelo do cliente, é recomendável adicionar vídeos e documentos bem.

![Seção Artefatos de Marketing da guia Detalhes da Vitrine](./media/marketing-artifacts-section.png)

Para concluir os campos nesta seção, use a tabela a seguir:
 
|    Campo                             |    DESCRIÇÃO                                                    |
|   -----------                        |    -------------                                                  |
| *Logos*                              |                                                                   |
| **Logotipo da oferta (formato png, 48x48)**   | Carregue um logotipo a ser exibido na visão geral do aplicativo ou nos resultados de pesquisa do aplicativo. AppSource dá suporte ao formato somente, PNG, com uma resolução de 48 x 48 px.  |
| **Logotipo de oferta (formato png, 216x216)** | Carregue um logotipo a ser exibido na página de detalhes do seu aplicativo.  AppSource dá suporte ao formato somente, PNG, com uma resolução de 216x216 px.  |
| *Vídeos*                             |                                                                   |
| **Nome**                             | Insira o nome ou o título do seu aplicativo.                                          |
| **URL**                              | Insira a URL de um vídeo hospedado no YouTube ou Vimeo.                              |
| **Miniatura**                        | Adicione uma imagem em miniatura do aplicativo.  AppSource dá suporte ao formato somente, PNG, com uma resolução de 1280 x 720 px.   |
| *Documentos*                          | Adicione até três documentos a serem exibidos no AppSource, sob o **Saiba mais** título.  |
| **Nome**                             | Insira o nome ou o título de um documento de suporte.                              |
| **Arquivo**                             | Carregue um arquivo PDF.                             |
| *Capturas de tela*                        | Adicione até cinco capturas de tela.                        |
| **Nome**                             | Insira um nome ou o título de uma captura de tela.                                       |
| **Imagem**                            | Carrega uma captura de tela PNG. Sua resolução deve ser 1280 x 720 px.  | 
|   |   |

Os logotipos que você carregar para [Portal do Cloud Partner](https://cloudpartner.azure.com) deve *não*:

- Use um gradiente. A aparência de seu logotipo deve ser simples.
- Inclua o nome da empresa, o nome da marca ou outro texto. 
- Procure ampliada.

## <a name="legal-section"></a>Seção legal

No **Legal** seção, fornecer os dois documentos legais necessários para cada oferta: sua política de privacidade e os termos de uso.

![Seção Jurídico da guia Detalhes da Vitrine](./media/legal-section.png)

Para concluir os campos nesta seção, use a tabela a seguir:

|   Campo                |   DESCRIÇÃO                           |
|------------------------|--------------------------------------   |
| **URL da política de privacidade** | URL para a política de privacidade postada       |
| **Termos de uso**       | Sua política de uso, formatados como texto sem formatação ou HTML simples     |
|  |  |


## <a name="customer-support-section"></a>Seção Atendimento ao Cliente

No **atendimento ao cliente** seção, forneça o **URL de suporte** para sua página de suporte do cliente online.  Esta página deve fornecer várias opções de contato, como telefone, email e bate-papo ao vivo. 


## <a name="lead-management-section"></a>Seção de gerenciamento de leads

No **gerenciamento de leads** seção, configurar seu sistema para coletar os clientes potenciais que geram suas ofertas do AppSource. Para escolher as opções de armazenamento para os clientes potenciais, use a tabela a seguir:

|    Campo               |   Destino do cliente potencial                               |
|------------------------|--------------------------------------            |
|  **Nenhum**              | Colete sem clientes potenciais. Essa é a opção padrão.  |
| **Blobs do Azure (preterido)** | Especificar [armazenamento de BLOBs do Azure](https://docs.microsoft.com/azure/storage/blobs/storage-blobs-overview) usando um nome de contêiner e uma cadeia de caracteres de conexão.  Essa opção é preterida. Use **tabelas do Azure** em vez disso.  |
| **Tabela do Azure**        | Especificar [armazenamento de tabelas do Azure](https://docs.microsoft.com/azure/cosmos-db/table-storage-overview) usando uma cadeia de conexão.  |
| **Dynamics CRM Online** | Especificar [Dynamics 365](https://dynamics.microsoft.com/) usando uma URL e credenciais de autenticação. |
| **Ponto de extremidade HTTPS**     | Especifique o ponto de extremidade HTTPS usando um conteúdo JSON.   |
| **Marketo**            | Especifique um [Marketo](https://www.marketo.com/) instância por meio de um servidor ID, ID do munchkin e ID do formulário.   |
| **Salesforce**         | Especificar [Salesforce](https://www.salesforce.com/) usando um identificador de objeto. |
|  |  |

Depois de publicar sua oferta, a conexão do cliente potencial é validada e um cliente potencial de teste é enviado automaticamente para o destino especificado. Gerenciar continuamente informações de cliente potencial e imediatamente, atualize suas configurações para refletir sua arquitetura de gerenciamento de cliente atual.
