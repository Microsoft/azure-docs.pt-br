# <a name="securing-external-collaboration-in-azure-active-directory-and-microsoft-365"></a>Protegendo a colaboração externa em Azure Active Directory e Microsoft 365

A colaboração segura com parceiros externos garante que os parceiros externos corretos tenham acesso apropriado a recursos internos durante o período certo. Por meio de uma abordagem de governança holística, você pode reduzir os riscos de segurança, atender às metas de conformidade e garantir que você sabe quem tem acesso.

A colaboração não controlada leva a uma falta de clareza na Propriedade do acesso e a possibilidade de exposição de recursos confidenciais. Migrar para colaboração segura e controlada pode garantir que haja linhas claras de propriedade e responsabilidade para acesso de usuários externos. Isso inclui:

* Gerenciar as organizações externas e os usuários dentro delas, que têm acesso aos recursos.

* Garantir que o acesso seja apropriado, revisado e associado quando for apropriado.

* Capacitando os proprietários de negócios a gerenciar a colaboração em trilhos de proteção criados pela ti.

Se você precisar atender às estruturas de conformidade, a colaboração governada permitirá que você atestar a adequação do acesso.

A Microsoft oferece conjuntos abrangentes de ferramentas para acesso externo seguro.  A colaboração B2B do Azure Active Directory (Azure AD) está no centro de qualquer plano de colaboração externo. O Azure AD B2B pode integrar-se a outras ferramentas no Azure AD e ferramentas no Microsoft 365 Services, para ajudar a proteger e gerenciar seu acesso externo.

Este conjunto de documentos foi criado para permitir que você passe da colaboração externa ad hoc ou sem controle flexível para um estado mais seguro.
